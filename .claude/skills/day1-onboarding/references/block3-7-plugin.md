## EXPLAIN

| 항목 | 내용 |
|------|------|
| 비유 | 앱스토어 — 다른 사람이 만든 스킬 패키지를 설치 |
| 핵심 | Skill + MCP + Hook + Agent 등을 **하나의 설치 단위**로 묶어서 공유 |
| 장점 | 팀 전체가 동일한 환경으로 일할 수 있다 |

### Plugin = 스킬의 묶음 배포

```
Plugin (앱스토어의 앱)
  ├── Skills (여러 스킬 포함)
  ├── MCP 설정 (외부 도구 연결)
  ├── Hooks (자동화 규칙)
  └── README (사용법)
```

### 설치 방법

```bash
# Claude Code에서 플러그인 설치
/install-plugin 플러그인URL
```

### 비유: 맥 앱 vs 앱스토어 앱

| 구분 | Skill (직접 만든 것) | Plugin (설치한 것) |
|------|---------------------|-------------------|
| 비유 | 직접 만든 요리 | 밀키트 주문 |
| 공유 | 파일 복사 | 한 줄 설치 |
| 업데이트 | 직접 수정 | 자동 업데이트 가능 |

> Plugin은 "다른 사람의 노하우를 한 줄로 설치하는 것"입니다. 이 캠프 자체도 Plugin으로 설치할 수 있습니다!

### 7개 기능 전체 구조도 (정리)

```
┌─ CLAUDE.md ─── Claude가 매번 읽는 "나에 대한 설명서"  [항상 로딩]
│
├─ Skill ─────── 반복 업무를 저장해둔 "레시피"           [필요할 때 로딩]
│   └─ MCP ───── 외부 도구 연결 "USB 포트"
│
├─ Subagent ──── 백그라운드 처리 (1:1 위임)              [병렬 처리]
│   └─ Agent Teams ── 팀 단위 처리 (N:N 협업)
│
├─ Hook ──────── 이벤트 기반 자동 실행                    [자동화]
│
└─ Plugin ────── 위의 모든 것을 패키지로 묶어 공유        [배포]
```

> 이 7개를 다 외울 필요는 없습니다. 가장 중요한 건 **CLAUDE.md** (항상 로딩되는 영구 기억)입니다.
