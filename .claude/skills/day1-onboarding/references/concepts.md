## 1. 지식노동자(Knowledge Worker)란?

### 정의
"생계를 위해 생각하는 사람" (Think for a living)
— Peter Drucker, 1959 『내일의 이정표(The Landmarks of Tomorrow)』

### 60년간의 핵심 역량

| 역량 | 설명 | 예시 |
|------|------|------|
| **패턴 인식** | 데이터에서 의미를 읽어냄 | 매출 데이터에서 계절 트렌드 발견 |
| **맥락적 사고** | 상황에 맞는 판단을 내림 | 같은 데이터도 업종마다 다르게 해석 |
| **메타인지** | 자기 사고 과정을 점검하고 교정 | "내가 이걸 모르는구나"를 아는 것 |

### 지능형 메모리의 합성 능력
- **200,000:1의 정보 압축**: 읽은 책, 경험, 교육을 핵심만 남기고 압축
- **전략적 제거**: 불필요한 정보를 버리고 중요한 것만 유지
- **멘탈 모델**: 체계적 의사결정을 위한 프레임워크

### 핵심 포인트
60년간 이 세 가지 역량 + 지능형 메모리가 지식노동자의 경쟁력이었습니다.

## 2. AI 시대의 변화 — "대체가 아니라 증폭"

### 2024년, AI가 지식노동 영역에 진입

| 역량 | AI의 수준 |
|------|----------|
| 패턴 인식 | AI가 인간보다 **빠르고 정확** |
| 맥락적 사고 | AI가 **빠르게 따라오는 중** |
| 메타인지 | 아직 인간의 영역... 하지만 **좁혀지는 중** |

### AI의 증폭 효과

| 지식노동자 능력 | × | AI의 증폭 |
|---------------|---|----------|
| 패턴 인식 | × | 무한한 데이터 처리 |
| 맥락적 사고 | × | 전체 파일 맥락 이해 |
| 메타인지 | × | 자율적 검증과 수정 |
| 전략적 직관 | × | 전 인류 경험의 합성 |
| 정보 압축 | × | 핵심 추출 자동화 |
| 멘탈 모델 | × | 프레임워크 즉시 적용 |

### 핵심 메시지
- 단순 덧셈이 아니라 **곱셈**
- 내 능력이 10이면 AI로 100배 → **1,000이 됨**
- "대체되는 게 아니라, 증폭되는 겁니다."

## 3. 도구의 진화 — 추상화 계층의 상승

### 비유
추상화 = 복잡한 것을 단순하게 감싸는 것. 에스컬레이터가 "걷기"를 추상화한 것처럼, 프로그래밍 도구도 점점 복잡함을 감춰왔습니다.

### 진화 과정

| 시대 | 도구 | 사용 가능 인원 | 특징 |
|------|------|--------------|------|
| 1950s | 기계어 | 수백 명 | 0과 1로 직접 대화 |
| 1960s | 어셈블리어 | 수천 명 | 기호로 한 단계 추상화 |
| 1970s | 고급 언어 | 수백만 명 | 인간의 논리에 가까운 문법 |
| 1990s | API | 수천만 명 | 기능 단위로 호출 |
| 2010s | SaaS | 수십억 명 | 클릭으로 소프트웨어 사용 |
| 2020s | **자연어** | **생각할 수 있는 모든 사람** | 말로 시키는 시대 |

### 로버트 라이시의 '상징 분석가' 4대 도구
1. **추상화** — 복잡한 것을 단순화
2. **시스템적 사고** — 전체를 보는 능력
3. **실험** — 가설을 세우고 검증
4. **협력** — 함께 문제 해결

→ 이 도구들이 이제 **자연어로 작동**합니다.

### 핵심 포인트
추상화 계층이 올라갈수록 더 많은 사람이 컴퓨터의 힘을 쓸 수 있게 되었습니다. 자연어 시대에는 **생각할 수 있는 모든 사람**이 사용자입니다.

## 4. Allocation Economy (할당 경제)

### 정의

| 과거 | 현재 |
|------|------|
| "**얼마나 잘** 하는가" = 생산성 | "**무엇을 시킬** 것인가" = 생산성 |

### 비유
- 과거: 직접 요리하는 셰프 → 얼마나 빠르고 맛있게 만드느냐
- 현재: 여러 셰프를 지휘하는 총괄 → 누구에게 어떤 요리를 맡길지 결정

### 핵심 포인트
- 내가 직접 처리하는 속도가 아니라, **AI에게 무엇을 시키고 내가 무엇에 집중할지** 결정하는 것이 경쟁력
- "할당"의 시대 — 자원(AI)을 어디에 배분할지 결정하는 능력

## 5. 지식노동자의 새로운 핵심 역량 3가지

### 과거 → 현재 변화

```
과거 핵심 역량: 패턴 인식 → 맥락적 사고 → 메타인지
       ↓ AI의 빠른 대체/증폭
새로운 핵심 역량: 맥락 정의 → 판단 기준 설정 → 결과 검증
```

### 상세 설명

| 새 역량 | 의미 | Claude Code 연결 | 캠프 학습일 |
|---------|------|-----------------|-----------|
| **맥락 정의** | 내 일의 본질을 AI가 이해할 수 있도록 구조화 | CLAUDE.md | Day 1 |
| **판단 기준 설정** | 멘탈 모델을 AI에게 이식하여 프레임워크 제공 | Skill, MCP | Day 2-3 |
| **결과 검증** | 메타인지로 AI 산출물의 질을 통제 | 워크플로우 완성 | Day 4 |

### 핵심 포인트
이 세 가지가 AI 시대에 지식노동자가 갖춰야 할 **근본적인 역량**입니다. 이번 캠프 전체가 이 세 역량을 훈련하는 과정입니다.

## 6. 7개 핵심 기능 비유 + 원리

### 전체 구조도

```
┌─ CLAUDE.md ─── Claude가 매번 읽는 "나에 대한 설명서"  [항상 로딩]
│
├─ Skill ─────── 반복 업무를 저장해둔 "레시피"           [필요할 때 로딩]
│   └─ MCP ───── 외부 도구 연결 "USB 포트"
│
├─ Subagent ──── 백그라운드 처리 (1:1 위임)              [병렬 처리]
│   └─ Agent Teams ── 팀 단위 처리 (N:N 협업)
│
├─ Hook ──────── 이벤트 기반 자동 실행                    [자동화]
│
└─ Plugin ────── 위의 모든 것을 패키지로 묶어 공유        [배포]
```

### 6-1. CLAUDE.md — "신입사원에게 주는 업무 매뉴얼"

**비유**: 신입사원 첫 출근 때 주는 매뉴얼
- "우리 팀은 이렇게 일해"
- "나는 이런 스타일을 좋아해"
- "이건 절대 하지 마"

**근본 원리**: 영구 기억 — 매번 새 대화를 시작해도 Claude가 기억하는 정보

**위치**:
- `./CLAUDE.md` — 프로젝트별 (이 프로젝트에서만 적용)
- `~/.claude/CLAUDE.md` — 전역 (모든 프로젝트에 적용)

### 6-2. Skill — "요리 레시피"

**비유**: 요리 레시피
- "이 재료들을 이 순서로 조합하면 이 요리가 나와"
- 재료와 순서만 따라하면 **같은 결과**

**근본 원리**: 점진적 로딩 (Just-in-time)
- CLAUDE.md는 **항상** 로딩
- Skill은 **필요할 때만** 로딩 (호출하면 그때 읽음)

**구조**:
```
.claude/skills/
└── my-skill/
    ├── SKILL.md      (필수 — 레시피 본문)
    ├── scripts/       (자동화 스크립트)
    ├── references/    (참조 자료)
    └── assets/        (이미지 등)
```

### 6-3. MCP — "USB-C 포트"

**비유**: USB-C 포트
- USB-C 하나로 충전기, 모니터, 외장하드 다 연결
- MCP 하나로 Slack, Notion, Gmail 다 연결

**근본 원리**: AI가 외부 서비스와 대화하는 **표준 규약** (Model Context Protocol)

**작동 방식**: 사용자 → Claude → MCP 서버 → API 호출 → 결과 반환

**연결 가능한 도구**:
- 협업: Slack, Notion, Linear, Asana, JIRA
- 이메일: Gmail, Outlook
- 캘린더: Google Calendar
- 개발: GitHub, GitLab, Sentry
- 디자인: Figma
- 데이터: PostgreSQL, BigQuery

### 6-4. Subagent — "부하 직원에게 일 맡기기"

**비유**: "이거 해놓고 끝나면 알려줘" → 나는 다른 일 하고 있으면 됨

**근본 원리**: 독립된 컨텍스트에서 **병렬 실행**

```
메인 Claude ─┬─ Subagent 1 (리서치)
             ├─ Subagent 2 (분석)
             └─ Subagent 3 (요약)
→ 각자 독립적으로 일하고, 결과만 보고
```

### 6-5. Agent Teams — "프로젝트 TF"

**Subagent와의 차이**:

| 구분 | Subagent | Agent Teams |
|------|----------|-------------|
| 관계 | 1:1 위임 | 팀 프로젝트 |
| 소통 | 보고만 받음 | 팀원 간 소통 가능 |
| 작업 | 독립적 | 공유 태스크 |

### 6-6. Hook — "IFTTT / 자동 체크리스트"

**비유**: "파일이 저장되면 → 자동으로 포맷 정리", "커밋 전에 → 자동으로 테스트 실행"

**근본 원리**: 결정론적 자동화 — 특정 이벤트가 발생하면 **정해진 동작**을 확실히 실행

| 이벤트 (trigger) | 동작 (action) |
|-----------------|--------------|
| PreToolUse | 도구 사용 전 체크 |
| PostToolUse | 도구 사용 후 정리 |
| Notification | 알림 보내기 |
| Stop | 세션 종료 시 정리 |

### 6-7. Plugin — "앱스토어 / 종합 패키지"

**비유**: 앱스토어에서 앱 하나 설치하면 관련 기능이 다 딸려오는 것

**원리**: Skill + MCP + Hook + Agent 등을 **하나의 설치 단위**로 묶어서 팀 전체가 동일 환경 사용

```
# 개별 설치 (Plugin 없이)    vs    Plugin (한 번에)
Skill A ← 수동 복사                marketing-plugin
Skill B ← 수동 복사                 ├─ Skill A, B
MCP 설정 ← 수동 설정                ├─ MCP 설정
Hook 설정 ← 수동 설정               ├─ Hook 설정
팀원 각자 반복...                    └─ Agent
                                   claude plugin add ← 한 줄
```

## 7. CLI 핵심 명령어 5개

### 비유: 터미널 = 텍스트로 컴퓨터랑 대화하는 카카오톡

| 명령어 | 의미 | 비유 |
|--------|------|------|
| `cd 폴더명` | 폴더 이동 | 파인더에서 폴더 더블클릭 |
| `ls` | 목록 보기 | 폴더 열어서 뭐 있나 보기 |
| `pwd` | 현재 위치 | 주소창 보기 |
| `cat 파일명` | 파일 내용 보기 | 메모장으로 파일 열어보기 |
| `open 파일명` | 파일/폴더 열기 | 파인더에서 더블클릭 (Mac) |

### 핵심 포인트
- 이 5개만 알면 충분
- 나머지는 Claude가 다 해줌
- 명령어를 몰라도 **말로 시키면** 됨

## 8. git / GitHub 기초

### git — "실행 취소의 끝판왕"

| 동작 | 비유 | 설명 |
|------|------|------|
| `git clone` | Google Drive에서 폴더 다운로드 | 원격 저장소를 내 컴퓨터로 복사 |
| `git commit` | 게임 세이브 | 현재 상태를 저장 (언제든 돌아갈 수 있음) |
| `git branch` | A안, B안 동시 작업 | 분기를 만들어 독립적으로 작업 |
| `git push` | Google Drive에 업로드 | 내 저장을 원격에 올리기 |

### GitHub — "코드의 Google Drive"

| 개념 | 비유 | 설명 |
|------|------|------|
| Repository | Google Drive 폴더 | 프로젝트 파일들이 모여있는 곳 |
| Pull Request (PR) | "이렇게 바꿨는데 괜찮아?" | 변경사항 리뷰 요청 |
| Merge | "오케이, 합치자" | 리뷰 통과 후 반영 |

### 핵심 포인트
- git = **뭘 해도 돌아갈 수 있음** (실험을 두려워하지 마세요)
- GitHub = **협업의 허브** (혼자가 아니라 함께)

## 9. 비개발자 FAQ

### Q: 터미널이 무서워요
"터미널은 텍스트로 컴퓨터랑 대화하는 창입니다. 카카오톡처럼요. Claude가 대신 명령어를 입력해주니까, 여러분은 말로 시키기만 하면 됩니다."

### Q: 명령어를 다 외워야 하나요?
"아니요! Claude에게 '현재 폴더 알려줘'라고 하면 알아서 pwd를 실행합니다. **모르면 Claude에게 물어보면 됩니다.**"

### Q: 잘못 입력하면 컴퓨터가 망가지나요?
"git을 쓰면 언제든 이전 상태로 돌아갈 수 있습니다. 게임 세이브처럼요. 실험을 두려워하지 마세요."

### Q: 코딩을 배워야 하나요?
"코딩 문법을 외울 필요 없습니다. Claude가 코드를 작성하고, 여러분은 지시하면 됩니다. 중요한 건 **무엇을 시킬지 결정하는 것**입니다. 이것이 바로 'Allocation Economy'입니다."

### Q: Claude Code와 웹의 Claude는 뭐가 다른가요?
"웹의 Claude는 브라우저에서 대화만 합니다. Claude Code는 내 컴퓨터에서 직접 돌아가면서 파일을 보고, 만들고, 수정합니다. **Claude가 내 컴퓨터에 출근해서 일하는 것**이라고 생각하세요."

### Q: 왜 GUI 도구가 아니라 터미널인가요?
"GUI는 개발자가 미리 만들어둔 버튼만 누를 수 있지만, CLI는 **제한이 없습니다**. AI 에이전트가 CLI를 통해 컴퓨터를 직접 제어할 수 있고, 모두가 코딩으로 새로운 차원의 일을 하게 됩니다."
