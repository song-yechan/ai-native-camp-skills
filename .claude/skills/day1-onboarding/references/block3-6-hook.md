## EXPLAIN

| 항목 | 내용 |
|------|------|
| 비유 | IFTTT / 자동 체크리스트 — "~하면 ~해라" |
| 핵심 | 특정 이벤트 발생 시 **자동으로** 정해진 동작을 실행 |
| 특징 | AI의 판단이 아닌 **결정론적(확실한)** 실행 |

### Hook 이벤트 종류

| 이벤트 | 언제 실행 | 예시 |
|--------|----------|------|
| PreToolUse | 도구 사용 **전** | "파일 저장 전에 포맷 체크" |
| PostToolUse | 도구 사용 **후** | "파일 저장 후 자동 포맷팅" |
| Notification | 알림 발생 시 | "에러 발생 시 슬랙 알림" |
| Stop | 세션 종료 시 | "세션 종료 시 작업 기록 저장" |

### 비유: 식당 자동 시스템

```
주문 들어옴 (이벤트)
  │
  ├── PreToolUse: 재료 재고 확인 (실행 전 체크)
  │
  ├── [요리 실행]
  │
  ├── PostToolUse: 접시 세팅 + 벨 울리기 (실행 후 정리)
  │
  └── Stop: 마감 시 매출 정산 (세션 종료 시)
```

> Hook은 AI가 "할까 말까 판단"하는 게 아니라, 조건이 맞으면 **무조건 실행**합니다. 이것이 "결정론적"의 의미입니다.
