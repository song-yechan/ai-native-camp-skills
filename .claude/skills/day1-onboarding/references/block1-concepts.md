## EXPLAIN

### 지식노동자(Knowledge Worker)란?

Peter Drucker가 1959년에 만든 개념입니다. 한마디로 **"생계를 위해 생각하는 사람"**.

60년간 지식노동자의 3대 핵심 역량:
- **패턴 인식** — 데이터에서 의미를 읽어냄
- **맥락적 사고** — 상황에 맞는 판단
- **메타인지** — 자기 사고 과정을 점검

**지능형 메모리의 합성 능력:**
- 200,000:1의 정보 압축 — 읽은 책, 경험, 교육을 핵심만 남기고 압축
- 전략적 제거 — 불필요한 정보를 버리고 중요한 것만 유지
- 멘탈 모델 — 체계적 의사결정을 위한 프레임워크

### 2024년, AI가 이 영역에 진입했습니다

| 역량 | AI의 수준 |
|------|----------|
| 패턴 인식 | 인간보다 빠르고 정확 |
| 맥락적 사고 | 빠르게 따라오는 중 |
| 메타인지 | 아직 인간의 영역... 하지만 좁혀지는 중 |

**핵심: AI는 대체가 아니라 "증폭"입니다.**

| 지식노동자 능력 | × | AI의 증폭 |
|---------------|---|----------|
| 패턴 인식 | × | 무한한 데이터 처리 |
| 맥락적 사고 | × | 전체 파일 맥락 이해 |
| 메타인지 | × | 자율적 검증과 수정 |
| 전략적 직관 | × | 전 인류 경험의 합성 |

내 능력이 10이면, AI로 100배 → **1,000이 됩니다.** 단순 덧셈이 아니라 **곱셈**입니다.

### 도구의 진화 — 추상화 계층의 상승

추상화 = 복잡한 것을 단순하게 감싸는 것. 에스컬레이터가 "걷기"를 추상화한 것처럼, 프로그래밍 도구도 점점 복잡함을 감춰왔습니다.

| 시대 | 도구 | 사용 가능 인원 |
|------|------|--------------|
| 1950s | 기계어 | 수백 명 |
| 1970s | 고급 언어 | 수백만 명 |
| 2010s | SaaS | 수십억 명 |
| **2020s** | **자연어** | **생각할 수 있는 모든 사람** |

### Allocation Economy (할당 경제)

과거: "얼마나 **잘** 하는가" = 생산성
현재: "**무엇을 시킬** 것인가" = 생산성

직접 요리하는 셰프 → 여러 셰프를 지휘하는 **총괄**로 역할이 바뀌고 있습니다.

### 지식노동자의 새로운 핵심 역량 3가지

```
과거 핵심 역량: 패턴 인식 → 맥락적 사고 → 메타인지
       ↓ AI의 빠른 대체/증폭
새로운 핵심 역량: 맥락 정의 → 판단 기준 설정 → 결과 검증
```

| 새 역량 | 의미 | Claude Code 연결 | 캠프 학습일 |
|---------|------|-----------------|-----------|
| **맥락 정의** | 내 일의 본질을 AI가 이해할 수 있도록 구조화 | CLAUDE.md | Day 1 |
| **판단 기준 설정** | 멘탈 모델을 AI에게 이식하여 프레임워크 제공 | Skill, MCP | Day 2-3 |
| **결과 검증** | 메타인지로 AI 산출물의 질을 통제 | 워크플로우 완성 | Day 4 |
