# Block 2: 스킬 제작 1 — 문제를 스킬 언어로 번역

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/skills
> ```

## EXPLAIN

### Step 1: 문제를 스킬 언어로 번역

자연어 문제 → **입력 → 처리 → 출력**으로 구조화

```
자연어: "뉴스레터가 너무 많아서 다 못 읽어요.
        중요한 인사이트만 정리하고 싶어요."

↓ 번역

스킬 언어:
  입력: Gmail에서 특정 키워드가 포함된 이메일
  처리: 각 이메일에서 핵심 인사이트 추출
  출력: insights/YYYY-MM-DD.md 파일
```

**팁**: "입력 → 처리 → 출력"으로 정리하면 스킬이 보입니다.

## EXECUTE

**Step 0: 암묵지 발굴**

스킬로 만들기 전에, 먼저 내 업무 속 숨은 노하우를 꺼내보세요. 아래 3가지 질문에 답해보세요:

1. **"이 업무를 후임에게 인수인계한다면 반드시 알려줘야 할 것은?"**
2. **"이 업무에서 나만의 판단 기준은? (다른 사람과 다르게 하는 것)"**
3. **"같은 일을 하는 동료가 나한테 자주 물어보는 것은?"**

> 💡 이 질문의 답변이 스킬의 "처리" 단계에 들어갈 핵심 로직이 됩니다.
> 예: "이메일 중요도는 제목에 특정 키워드가 있으면 높음" → 처리 단계에 "키워드 기반 중요도 분류" 로직 추가

**Step 1: 나만의 문제 번역하기**

스킬로 만들고 싶은 반복 업무나 불편한 점을 생각해보세요. 예를 들어:
- **뉴스레터/이메일 정리** — 이메일에서 핵심만 추출
- **미팅 안건 자동 생성** — 여러 도구에서 안건 수집
- **주간 리포트 작성** — 주간 업무 자동 요약
- 또는 나만의 아이디어를 자유롭게 입력

> 💡 Day 2에서 만든 Context Sync 스킬이 있다면, 그것을 개선하는 것도 좋은 선택입니다. 이미 v1이 있으니까요.

아이디어를 Claude에게 말하면, **입력 → 처리 → 출력**으로 번역하여 보여줍니다.

예시:
```
💡 번역 결과:

[원래 문제]
"매주 팀 미팅 전에 각 도구에서 진행 상황을 확인하고 안건을 정리하는 게 번거로워요"

[스킬 언어]
- 입력: 주간 미팅 날짜, 관련 도구 (Slack, Linear)
- 처리: 1. Slack에서 주요 논의 수집 2. Linear에서 진행 상태 수집 3. 안건 형식으로 정리
- 출력: meetings/YYYY-MM-DD-agenda.md
```

**Step 2: Clarify로 모호함 제거**

Claude에게 요청:
```
방금 번역한 스킬 계획에서 모호한 부분을 질문해줘
```

Claude의 질문에 답변하여 요구사항을 구체화한다.
