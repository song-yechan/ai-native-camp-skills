# Block 0: 스킬 설계 원칙

> 📖 공식 문서: [Claude Code Skills](https://docs.anthropic.com/en/docs/claude-code/skills)

## EXPLAIN

### 스킬 = AI에게 주는 업무 레시피

요리 레시피처럼:
- "이 재료들을 이 순서로 조합하면 이 요리가 나와"
- 재료와 순서만 따라하면 **누가 해도 같은 결과**

### CLAUDE.md vs Skill

| 구분 | CLAUDE.md | Skill |
|------|-----------|-------|
| 비유 | 업무 매뉴얼 | 요리 레시피 |
| 로딩 | **항상** (매 세션) | **필요할 때만** |
| 내용 | "나는 이런 사람" | "이 일은 이렇게" |
| 예시 | 선호 언어, 작업 스타일 | 주간 리포트 생성 |

**핵심**: 컨텍스트 윈도우는 유한 → 필요한 것만 로딩 (Just-in-time)

### 좋은 스킬의 3가지 조건

**1. 목적이 명확하다** — 한 문장으로 설명 가능
- 나쁜 예: "이메일 관련 여러 가지를 한다"
- 좋은 예: "Gmail에서 뉴스레터를 찾아 인사이트를 정리한다"

**2. 입력과 출력이 분명하다**
- 입력: 무엇을 받는가? (키워드? 파일? URL?)
- 출력: 무엇을 내놓는가? (md 파일? JSON? Slack 메시지?)

**3. 실행 단계가 구체적이다**
- 나쁜 예: "이메일을 적절히 정리한다"
- 좋은 예: "1. 키워드 검색 → 2. 상위 5개 선택 → 3. 핵심 추출 → 4. md 파일 생성"

> "적절히", "알아서" 같은 모호한 표현은 Claude가 매번 다르게 해석합니다. 구체적으로 적어야 일관된 결과를 얻습니다.

## QUIZ

**Q1. Skill이 "필요할 때만 로딩"되는 이유는?**

```json
{
  "questions": [{
    "question": "Skill이 '필요할 때만 로딩'되는 이유는?",
    "header": "Q1",
    "options": [
      {"label": "파일 크기가 너무 커서", "description": "용량 문제?"},
      {"label": "컨텍스트 윈도우가 유한하므로 효율적으로 사용하기 위해", "description": "Just-in-time 로딩"},
      {"label": "보안상의 이유로", "description": "보안?"},
      {"label": "Claude가 읽기 어려워서", "description": "읽기 난이도?"}
    ],
    "multiSelect": false
  }]
}
```
정답: 2번 (컨텍스트 윈도우가 유한)
해설: 컨텍스트 윈도우는 유한합니다. CLAUDE.md는 항상 필요하지만, Skill은 호출될 때만 필요합니다.

**Q2. 좋은 스킬의 3가지 조건이 아닌 것은?**

```json
{
  "questions": [{
    "question": "좋은 스킬의 3가지 조건이 아닌 것은?",
    "header": "Q2",
    "options": [
      {"label": "목적이 명확하다", "description": "한 문장으로 설명 가능"},
      {"label": "입력과 출력이 분명하다", "description": "무엇을 받고 무엇을 내놓는지"},
      {"label": "코드가 포함되어야 한다", "description": "코드 필수?"},
      {"label": "실행 단계가 구체적이다", "description": "모호함 없는 단계"}
    ],
    "multiSelect": false
  }]
}
```
정답: 3번 (코드가 포함되어야 한다)
해설: 코드 없이 SKILL.md 텍스트만으로도 스킬이 됩니다. 핵심은 목적, 입출력, 실행 단계입니다.
