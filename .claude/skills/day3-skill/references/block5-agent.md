## EXPLAIN

### 에이전트 = 팀원한테 일 나눠주기

Day 1에서 Subagent를 잠깐 소개했습니다. 이번에는 **실제로 활용하는 법**을 배웁니다.

**비유: 팀장의 업무 분배**

```
나 (팀장)
  │
  ├── 에이전트 1: "Slack 메시지 요약해와"     ← 동시에 실행
  ├── 에이전트 2: "캘린더 일정 정리해와"       ← 동시에 실행
  └── 에이전트 3: "이메일 확인해와"            ← 동시에 실행
  │
  ▼
  결과만 받아서 종합
```

- 내가 직접 하면: 하나씩 순서대로 처리 (느림)
- 에이전트에게 시키면: 여러 개를 **동시에** 처리 (빠름)

### 에이전트의 핵심 특징

| 특징 | 설명 | 비유 |
|------|------|------|
| **병렬 실행** | 여러 에이전트가 동시에 작업 | 팀원 3명이 각자 다른 일 |
| **독립된 맥락** | 각 에이전트는 메인 대화 맥락을 물려받지 않음 | 팀원한테 "이것만 해와"라고 시킨 것 |
| **결과만 반환** | 작업 과정은 에이전트 내부에서 처리, 결과만 돌려줌 | 팀원이 보고서만 제출 |

> "Blank Slate" — 에이전트는 깨끗한 상태에서 시작합니다. 그래서 **지시를 명확하게** 해야 합니다. (Clarify와 연결되죠!)

### 언제 에이전트를 쓰면 좋은가?

| 상황 | 에이전트 활용 | 이유 |
|------|-------------|------|
| 여러 소스에서 정보 수집 | 적합 | 각 소스를 동시에 조회 |
| 여러 파일을 각각 분석 | 적합 | 파일마다 에이전트 배정 |
| 앞 단계 결과가 다음 단계에 필요한 작업 | 부적합 | 순차적이라 병렬 불가 |
| 간단한 단일 작업 | 부적합 | 오히려 느려짐 (오버헤드) |

### Subagent vs Agent Teams — 두 가지 협업 방식

에이전트에는 두 가지 방식이 있습니다.

**Subagent = 팀원한테 각각 일 맡기고 결과 보고 받기**

```
나 (팀장)
  ├── 팀원A: "이거 조사해와" → 결과 보고
  ├── 팀원B: "저거 정리해와" → 결과 보고
  └── 팀원C: "이것도 확인해와" → 결과 보고

→ 각자 독립적으로 일하고, 나한테 결과만 보고
→ 팀원끼리는 대화하지 않음
```

**Agent Teams = 회의실에서 팀원들이 서로 논의하면서 일하기**

```
나 (팀장): "이 프로젝트를 함께 해결하자"
  ├── 팀원A (분석 담당): 데이터 분석
  ├── 팀원B (기획 담당): 기획안 작성
  └── 팀원C (검증 담당): 결과 검증

→ 팀원끼리 서로 소통하며 진행
→ A의 분석 결과를 B가 기획안에 반영
→ 공유 태스크 리스트로 협업
```

| 구분 | Subagent | Agent Teams |
|------|----------|-------------|
| 비유 | 팀원한테 일 맡기고 보고 받기 | 회의실에서 함께 일하기 |
| 관계 | 1:1 위임 (나 → 팀원) | 팀 프로젝트 (팀원끼리 소통) |
| 소통 | 결과 보고만 받음 | 에이전트 간 소통 가능 |
| 작업 | 독립적 (서로 모름) | 공유 태스크 (서로 앎) |
| 적합한 상황 | 독립적인 작업 여러 개 | 서로 연결된 복잡한 프로젝트 |

> 오늘은 **Subagent 방식**을 실습합니다. Agent Teams는 "이런 것도 가능하다" 정도만 알고 가세요.

## EXECUTE

### 실습 1: 에이전트에게 간단한 리서치 시키기

여러 파일을 **동시에** 요약하는 작업을 에이전트에게 시켜봅시다.

Claude에게 아래와 같이 요청하세요:

```
이 폴더에 있는 파일들을 각각 에이전트에게 보내서 동시에 요약해줘.
각 파일의 핵심 내용을 3줄로 정리해줘.
```

또는 더 구체적으로:

```
아래 3가지 작업을 각각 에이전트에게 동시에 시켜줘:
1. CLAUDE.md 파일 읽고 핵심 규칙 3개 정리
2. .claude/skills/ 폴더에 어떤 스킬들이 있는지 목록 정리
3. 최근 git 커밋 히스토리 5개 요약
```

→ Claude가 에이전트를 소환하여 병렬로 처리하는 모습을 관찰하세요.
→ 결과가 각각 돌아오는 것을 확인하세요.

### 실습 2: 에이전트에게 스킬 테스트 위임하기

Block 3에서 만든 나만의 스킬을 에이전트에게 테스트하라고 시켜봅시다.

```
에이전트를 하나 띄워서 내 [스킬이름] 스킬을 실행하고,
결과가 잘 나오는지 확인한 후 개선점을 3가지 알려줘.
```

→ 에이전트가 독립된 맥락에서 스킬을 실행하고 피드백을 줍니다.
→ "제3자가 내 스킬을 써보는 것"과 같은 효과입니다.

### 실습 3: 여러 에이전트 동시 실행

여러 도구에서 정보를 동시에 모아오는 작업을 시켜봅시다.

```
아래 3가지 작업을 각각 다른 에이전트에게 동시에 시켜줘:
1. Slack에서 오늘 내가 멘션된 메시지 요약
2. Google Calendar에서 이번 주 일정 정리
3. Gmail에서 읽지 않은 이메일 제목 5개 가져오기

결과를 하나의 "오늘의 브리핑"으로 합쳐줘.
```

→ 3개의 에이전트가 동시에 각각 다른 도구에 접근합니다.
→ 결과가 모이면 메인 Claude가 하나로 종합합니다.
→ 이것이 바로 "Context Sync + 에이전트"의 실전 활용입니다.

> MCP가 연결되어 있지 않은 도구는 에러가 날 수 있습니다. 연결된 도구로만 시도하세요.
