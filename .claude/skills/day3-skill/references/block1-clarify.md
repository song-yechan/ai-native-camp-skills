# Block 1: Clarify — 모호함을 명확함으로

> 이 블록은 Clarify 프로세스 설명이므로 공식 문서 URL이 없습니다.

## EXPLAIN

### Clarify = 모호함을 명확함으로 바꾸는 과정

```
Before: "이메일 정리해줘"
              ↓ Clarify
After:  "Gmail에서 'AI' 키워드 뉴스레터를 찾아서,
        핵심 인사이트를 요약하고,
        insights/YYYY-MM-DD.md로 저장해줘"
```

### 왜 필요한가?

**"Garbage In, Garbage Out"**
- 우리 머릿속 의도는 모호한 경우가 많음
- AI는 모호한 입력 → 추측해서 실행 → 높은 확률로 틀림
- Clarify로 **추측 대신 질문하게** 만들기

### 3가지 Clarify 스킬

| 스킬 | 용도 | 명령어 |
|------|------|--------|
| **Vague** | 모호함 해결 (가장 기본) | `/clarify-vague` |
| **Unknown** | 블라인드 스팟 찾기 | `/clarify-unknown` |
| **Meta** | 내용/형식 분리 검토 | `/clarify-meta` |

**Vague**: Claude가 모호한 부분을 찾아서 질문으로 명확화
**Unknown**: "아는 것 / 모르는 것 / 모르는 것을 모르는 것" 4분면 분석으로 블라인드 스팟 발견
**Meta**: 만들려는 내용(Content)과 담는 형식(Form)을 분리해서 검토

### 최적의 타이밍: 플래닝 직후!

```
1. 요청 입력
2. Claude가 계획을 만듦 (Shift+Tab)
3. ⭐ 여기서 Clarify! ⭐
4. 계획 수정 → 실행
```

**왜 플래닝 직후인가?**
- Claude는 내 입력을 "참"이라고 가정하고 플래닝
- 모호한 입력 그대로 실행하면 모호한 결과
- 플랜이 만들어지자마자 Clarify로 수정하는 게 가장 효율적

## EXECUTE

**Clarify 체험**

아래 모호한 요청을 Claude에 입력해보세요:
```
우리 팀 업무를 개선해줘
```

Claude가 바로 실행하려 할 것입니다. 이때 아래를 입력하세요:
```
잠깐, 모호한 부분을 먼저 질문해줘. 어떤 팀인지, 어떤 업무인지, 개선 기준이 뭔지 등.
```

→ Claude가 Clarify 질문을 하기 시작합니다. 질문에 답하면서 요구사항이 구체화되는 과정을 체험하세요.
