# Day 3: 스킬 설계와 제작

이 스킬이 호출되면 아래 **STOP PROTOCOL**을 반드시 따른다.

---

## 용어 정리

이 스킬에서 사용하는 핵심 용어:

| 용어 | 설명 |
|------|------|
| **스킬(Skill)** | Claude에게 특정 작업 방법을 가르치는 문서. `/스킬이름`으로 호출. "레시피처럼 단계가 적힌 매뉴얼" |
| **SKILL.md** | 스킬의 본체 파일. `.claude/skills/스킬이름/SKILL.md` 경로에 저장 |
| **frontmatter** | 스킬 파일 맨 위에 `---`로 감싸서 적는 "이름표". name과 description을 여기에 적는다 |
| **Clarify** | 모호한 요구사항을 명확하게 만드는 과정. "뭘 만들지 확실히 하고 시작하기" |
| **Vague** | 두루뭉술한 상태. "번역해줘" → 무슨 언어? 어떤 톤? 어디에 저장? |
| **스킬 체이닝** | 스킬A의 출력을 스킬B의 입력으로 연결하는 것. "공장 생산라인처럼 한 공정씩" |
| **플랜 모드** | `Shift+Tab`으로 진입. 실행 전에 계획을 세우는 모드 |

## STOP PROTOCOL — 절대 위반 금지

> 이 프로토콜은 이 스킬의 최우선 규칙이다.
> 아래 규칙을 위반하면 수업이 망가진다.

### 각 블록은 반드시 2턴에 걸쳐 진행한다

```
┌─ Phase A (첫 번째 턴) ──────────────────────────────┐
│ 1. references/에서 해당 블록 파일의 EXPLAIN 섹션을 읽는다    │
│ 2. 기능을 설명한다                                        │
│ 3. references/에서 해당 블록 파일의 EXECUTE 섹션을 읽는다    │
│ 4. "지금 직접 실행해보세요"라고 안내한다                     │
│ 5. ⛔ 여기서 반드시 STOP. 턴을 종료한다.                    │
│                                                          │
│ ❌ 절대 하지 않는 것: 퀴즈 출제, QUIZ 섹션 읽기             │
│ ❌ 절대 하지 않는 것: AskUserQuestion 호출                  │
│ ❌ 절대 하지 않는 것: "실행해봤나요?" 질문                   │
└──────────────────────────────────────────────────────────┘

  ⬇️ 사용자가 돌아와서 "했어", "완료", "다음" 등을 입력한다

┌─ Phase B (두 번째 턴) ──────────────────────────────┐
│ 1. references/에서 해당 블록 파일의 QUIZ 섹션을 읽는다       │
│ 2. AskUserQuestion으로 퀴즈를 출제한다                     │
│ 3. 정답/오답 피드백을 준다                                 │
│ 4. 다음 블록으로 이동할지 AskUserQuestion으로 묻는다         │
│ 5. ⛔ 다음 블록을 시작하면 다시 Phase A부터.                │
└──────────────────────────────────────────────────────────┘
```

### 핵심 금지 사항 (절대 위반 금지)

1. **Phase A에서 AskUserQuestion을 호출하지 않는다** — 설명 + 실행 안내 후 바로 Stop
2. **Phase A에서 퀴즈를 내지 않는다** — QUIZ 섹션은 Phase B에서만 읽는다
3. **Phase A에서 "실행해봤나요?"를 묻지 않는다** — 사용자가 먼저 말할 때까지 기다린다
4. **한 턴에 EXPLAIN + QUIZ를 동시에 하지 않는다** — 반드시 2턴으로 나눈다

### 공식 문서 URL 출력 (절대 누락 금지)

모든 블록의 Phase A 시작 시, 해당 reference 파일 상단의 `> 공식 문서:` URL을 **반드시 그대로 출력**한다.

```
📖 공식 문서: [URL]
```

- reference 파일에 URL이 여러 개 있으면 전부 출력한다
- URL을 요약하거나 생략하지 않는다

### Phase A 종료 시 필수 문구

Phase A의 마지막에는 반드시 아래 형태의 문구를 출력하고 Stop한다:

```
---
👆 위 내용을 직접 실행해보세요.
실행이 끝나면 "완료" 또는 "다음"이라고 입력해주세요.
```

이 문구 이후에 어떤 도구 호출(AskUserQuestion 포함)이나 추가 텍스트도 출력하지 않는다.

## 핵심 전략: 모호함 제거 → 스킬 제작 → 체이닝

아래 방식으로 진행한다:

1. Block 0에서 좋은 스킬의 3가지 조건을 이해한다
2. Block 1에서 Clarify로 모호한 요구사항을 명확하게 만든다
3. Block 2에서 문제를 "입력 → 처리 → 출력"으로 번역한다
4. Block 3에서 SKILL.md를 직접 작성하고 테스트한다
5. Block 4에서 스킬 체이닝 개념을 배우고 설계한다
6. Block 5에서 에이전트를 활용하여 병렬 작업과 스킬 테스트를 경험한다

> 핵심은 Block 3에서 **실제로 동작하는 스킬을 만드는 것**이다. 나머지는 이를 위한 준비 과정.

## 블록 특수 규칙

- **Block 0 (스킬 설계 원칙)**: Phase A에서 좋은 스킬의 3조건(명확성, 입출력, 구체성) 설명 → Stop. Phase B에서 퀴즈.
- **Block 1 (Clarify)**: QUIZ가 없다. Phase A에서 Vague/Unknown/Meta 3가지 모호함 유형 설명 + 실습 안내 → Stop. 완료 확인 후 다음 블록.
- **Block 2 (스킬 제작 1)**: QUIZ가 없다. Phase A에서 문제를 스킬 언어로 번역하는 방법 안내 → Stop. 완료 확인 후 다음 블록.
- **Block 3 (스킬 제작 2)**: Phase A에서 SKILL.md 필수 구조 설명 + 참가자가 직접 작성하도록 안내 → Stop. Phase B에서 퀴즈. (가장 긴 블록 — 완료 후 "여기까지 잘 따라오셨습니다!" 격려)
- **Block 4 (스킬 체이닝)**: Phase A에서 체이닝 개념 + 설계 실습 안내 → Stop. Phase B에서 퀴즈.
- **Block 5 (에이전트 활용)**: Phase A에서 에이전트 개념(팀원한테 일 나눠주기) + 3가지 실습 안내 → Stop. Phase B에서 퀴즈.

## 진행 규칙

- 한 번에 한 블록씩 진행한다
- "다음", "skip", 블록 번호/이름으로 이동한다
- Block 3에서 생성한 스킬 파일은 참가자의 프로젝트 `.claude/skills/` 아래에 저장한다
- 각 블록 시작 시 진행 상태를 표시한다: `📍 Day 3 진행: [Block N/5] — 블록 제목`
- 모든 응답은 **한국어**로 한다
- Claude Code 관련 질문이 오면 claude-code-guide 에이전트(내장 도구)로 답변한다. 답변 후 사용자가 직접 따라할 수 있게 단계별로 안내하고, 질문할 때는 AskUserQuestion을 사용한다. 내장 에이전트 답변이 부정확하다고 판단되면, 공식 문서를 `curl`로 파일에 저장한 뒤 Read 툴로 꼼꼼히 읽고 정확한 정보로 다시 답한다 (WebFetch는 요약/손실 위험이 있으므로 사용하지 않는다)
