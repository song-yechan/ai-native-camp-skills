## 신규 추가 — 섹션 5: 에이전트 활용 (20분)

### 슬라이드 5-1: 섹션 타이틀
- "에이전트 활용 — 팀원한테 일 나눠주기"

### 슬라이드 5-2: 에이전트란?
- 에이전트 = Claude가 다른 Claude를 불러서 일을 시킴
- 비유: 팀장의 업무 분배 (에이전트 3개가 Slack/Calendar/Email 동시 처리)
- Day 1 Subagent 소개 연결

### 슬라이드 5-3: 에이전트의 핵심 특징
- ① 병렬 실행 ② 독립된 맥락 (Blank Slate) ③ 결과만 반환
- Blank Slate이므로 Clarify에서 배운 "명확한 지시"가 중요

### 슬라이드 5-4: Subagent vs Agent Teams
- Subagent = 팀원한테 각각 일 맡기고 결과 보고 받기 (독립 작업)
- Agent Teams = 회의실에서 팀원들이 서로 논의하면서 일하기 (소통 협업)
- "오늘은 Subagent를 실습합니다. Agent Teams는 소개만"

### 슬라이드 5-5: 언제 쓰면 좋은가?
- ✅ 적합: 여러 소스 동시 수집, 여러 파일 분석, 스킬 테스트
- ❌ 부적합: 순차 작업(→ 체이닝), 간단한 단일 작업

### 슬라이드 5-6: 실습 안내
- 실습 1: 에이전트에게 리서치 시키기 (파일 3개 동시 요약)
- 실습 2: 에이전트에게 스킬 테스트 위임
- 실습 3: 여러 에이전트 동시 실행 (Slack + Calendar + Gmail)
- ⏱️ 10분간 개인 실습

### 슬라이드 5-7: 에이전트 vs 체이닝 vs 직접 실행
- 직접 실행: 하나씩 순서대로
- 스킬 체이닝: 결과를 순서대로 이어붙이기 (순차)
- 에이전트: 여러 팀원에게 동시에 나눠주기 (병렬)

### 슬라이드 5-8: 섹션 5 정리
