# Day 1: "이게 되네?" — 첫 경험 + 문제 발굴

## 1. Day 1 개요

| 항목 | 내용 |
|------|------|
| 테마 | 발견 (Discovery) |
| 핵심 메시지 | AI는 도구가 아니라 파트너다. 내 업무에서 AI로 해결할 수 있는 문제를 찾아라. |
| 총 소요 시간 | 약 150분 (2시간 30분) |
| 사전 과제 | Claude Code 설치 완료 + Anthropic 계정 연동 |
| 산출물 | 문제 정의서 (각자 해결하고 싶은 문제 1개 이상) + 첫 CLAUDE.md |

### 학습 목표

1. Claude Code 환경에서 대화하며 작업할 수 있다
2. CLI 기본 명령어 5개를 이해하고 활용할 수 있다
3. 7개 핵심 기능의 역할과 관계를 이해한다
4. 자신의 업무에서 AI로 해결할 문제를 1개 이상 정의한다

### 시간 배분

| 섹션 | 시간 | 내용 |
|------|------|------|
| 섹션 1 | 20분 | 오프닝 + 참여자 소개 |
| 섹션 2 | 30분 | 환경 세팅 (설치 확인 + 첫 대화 + CLI 기초) |
| 섹션 3 | 50분 | 7개 핵심 기능 |
| 섹션 4 | 35분 | 문제 발굴 워크숍 |
| 섹션 5 | 15분 | 과제 설정 + 마무리 |

### 섹션 2: 환경 세팅 (30분)

#### 슬라이드 2-1: 섹션 타이틀
```
Claude Code 첫 만남

```

#### 슬라이드 2-2: AI 시대의 변화
```
AI 시대, 지식 노동의 본질이 바뀌고 있다

과거의 지식 노동자          AI 시대의 지식 노동자
─────────────────────────────────────────────────
"내가 직접 한다"      →    "무엇을 시킬 것인가"를 결정한다
엑셀 함수를 외운다    →    AI에게 맥락을 설계한다
코드를 직접 짠다      →    결과를 검증하고 방향을 조정한다

핵심 역량의 변화:
"How to do" → "What to do" → "What to delegate"
```

**발표자 노트**:
- 과거에는 "어떻게 할 것인가(How)"가 핵심 역량
- AI 시대에는 "무엇을 시킬 것인가(What to delegate)"가 핵심
- 이 캠프에서 배우는 것이 바로 이것

#### 슬라이드 2-3: 도구의 진화
```
도구의 진화: 추상화 계층의 상승

펀치카드   →   키보드   →   GUI   →   자연어
  │            │          │         │
  │            │          │         └─ "이거 해줘"
  │            │          └─────────── 마우스 클릭
  │            └────────────────────── 타이핑
  └─────────────────────────────────── 구멍 뚫기

추상화 계층이 올라갈수록 → 진입 장벽 하락
"이제 코딩을 몰라도 AI와 함께 일할 수 있다"
```

#### 슬라이드 2-4: Claude Code란?
```
Claude Code = AI 시대의 업무 운영체제(OS)

┌─────────────────┬─────────────────────────────┐
│   컴퓨터 OS     │      Claude Code (AI OS)    │
├─────────────────┼─────────────────────────────┤
│ 하드웨어 제어   │ 도구 제어 (Slack, Notion 등)│
│ 앱 실행        │ 워크플로우 실행              │
│ 파일 관리      │ 맥락 관리                   │
└─────────────────┴─────────────────────────────┘

컴퓨터 OS가 하드웨어를 제어하듯,
Claude Code는 AI 환경에서 도구를 제어하고 워크플로우를 실행합니다.
```

**발표자 노트**:
- Claude Code는 단순한 챗봇이 아님
- AI와 함께 일하는 "운영체제"
- 오늘 세 가지를 체험: 도구 제어, 워크플로우 실행, 맥락 관리

#### 슬라이드 2-8: CLI가 뭔가요?
```
CLI가 뭔가요?

CLI = Command Line Interface
텍스트로 컴퓨터와 대화하는 방식

비유: "터미널 = 텍스트 카카오톡"

GUI (그래픽)          CLI (텍스트)
─────────────────────────────────────
폴더 더블클릭    →    cd 폴더명
파일 목록 보기  →    ls
주소창 보기     →    pwd
```

**발표자 노트**:
- 터미널이 무섭지 않다는 것을 강조
- GUI에서 하던 것을 텍스트로 하는 것일 뿐

#### 슬라이드 2-9: CLI, 왜 알아야 하나?
```
CLI, 왜 알아야 하나?

"Claude가 다 해주는데 왜 알아야 해요?"

→ 기본을 알면 더 잘 시킬 수 있습니다

비유: 요리사에게 주문할 때
❌ "맛있는 거 주세요"
✅ "해산물 파스타인데, 매운 건 빼주세요"

Claude에게도 마찬가지입니다.
뒤에서 뭘 하는지 알면 더 정확하게 시킬 수 있습니다.
```

#### 슬라이드 2-10: CLI 핵심 명령어 5개
```
CLI 핵심 명령어 5개

┌─────────┬─────────────────┬──────────────────────────┐
│ 명령어  │ 의미            │ 비유                     │
├─────────┼─────────────────┼──────────────────────────┤
│ cd      │ 폴더 이동       │ 파인더에서 폴더 더블클릭 │
│ ls      │ 목록 보기       │ 폴더 열어서 뭐 있나 보기 │
│ pwd     │ 현재 위치       │ 주소창 보기              │
│ cat     │ 파일 내용 보기  │ 메모장으로 파일 열기     │
│ open    │ 파일/폴더 열기  │ 더블클릭 (Mac)           │
└─────────┴─────────────────┴──────────────────────────┘

💡 이 5개만 알면 충분합니다. 나머지는 Claude가 다 해줍니다.
```

#### 슬라이드 2-11: CLI 실습 1

CLI 실습 1: Claude에게 시켜보기

#### 슬라이드 2-12: CLI 실습 2
```
CLI 실습 2: 직접 실행해보기

Claude Code 안에서 ! 를 붙이면 직접 실행됩니다

!pwd       → 현재 위치 확인
!ls        → 파일 목록 보기
!open .    → 현재 폴더를 Finder로 열기

💡 ! = "Claude한테 시키지 말고 내가 직접 실행할게"
```

**발표자 노트**:
- !open . 명령어는 굉장히 유용함 - 파인더가 바로 열림
- cat으로 파일 내용 확인하면 Claude의 요약 없이 원문 그대로 볼 수 있음

#### 슬라이드 3-2: 7개 기능 전체 구조
```
7개 기능의 관계

┌─ CLAUDE.md ─── Claude가 매번 읽는 "나에 대한 설명서"
│
├─ Skill ─────── 반복 업무를 저장해둔 "레시피"

├─ Subagent ──── Claude한테 일 시키는 것 (백그라운드에서 알아서 처리)
│   └─ Agent Teams ── 여러 명한테 동시에 일 시키기
│
├─ Hook ──────── "이런 일이 생기면 자동으로 이거 해줘"
│
└─ Plugin ────── 위에 만든 것들을 팀원한테 공유하는 패키지

📖 공식 문서: https://code.claude.com/docs/ko/features-overview
```

**발표자 노트**: 
- 이 구조를 먼저 보여주고, 하나씩 설명

#### 슬라이드 3-4: CLAUDE.md 다이어그램
```
CLAUDE.md 작동 방식

세션 시작
    │
    ▼
┌─────────────┐     ┌─────────────┐
│ CLAUDE.md   │────▶│ 시스템       │────▶ 규칙을 아는 상태로
│ (영구 기억)  │     │ 프롬프트     │      대화 시작
└─────────────┘     └─────────────┘
  항상 읽힘            매번 자동 주입

왜 중요한가?
- AI는 기본적으로 "휘발성 기억"을 가짐 (대화가 끝나면 잊음)
- CLAUDE.md는 이 휘발성 기억을 "영구 기억"으로 만들어줌
- 매번 같은 말을 반복할 필요가 없음
```

### 3.5 Subagent vs Agent Teams

```
[Subagent vs Agent Teams 비교]

1. Subagent
   정의: 메인 Claude가 다른 Claude를 불러서 작업 위임
   비유: 부하 직원 (1:1 위임, 보고만)
   
   특징:
   - 독립된 컨텍스트 (Blank Slate)
   - 메인 맥락을 물려받지 않음
   - 작업 완료 후 요약만 반환
   - 병렬 실행 가능
   
   적합한 작업:
   - 읽기(Read) 작업이 많은 경우
   - 여러 소스에서 정보 수집
   - 리서치, 분석
   
   예시:
   "Slack, Gmail, Calendar 동시에 수집해줘"
   → 3개의 Subagent가 각각 수집
   → 결과만 메인에게 전달

2. Agent Teams
   정의: 여러 AI가 팀처럼 협업
   비유: 프로젝트 팀 (팀원 간 소통 + 공유 태스크)
   
   특징:
   - 팀원 간 메시지 전송 가능
   - 공유 태스크 리스트
   - 더 많은 토큰 사용
   - 현재 실험적 기능
   
   적합한 작업:
   - 다각적 분석이 필요한 경우
   - 여러 관점의 검토
   - 복잡한 협업 워크플로우

3. 언제 뭘 쓸까?
   ┌─────────────────────┬─────────────────────┐
   │ Subagent            │ Agent Teams         │
   ├─────────────────────┼─────────────────────┤
   │ 독립적 작업         │ 협업이 필요한 작업  │
   │ 정보 수집           │ 다각적 분석         │
   │ 단순 병렬 처리      │ 복잡한 워크플로우   │
   │ 토큰 효율 중요      │ 품질이 더 중요      │
   └─────────────────────┴─────────────────────┘
```

### 3.6 CLI 핵심 명령어 상세

```
[CLI 핵심 명령어 상세]

1. cd (Change Directory)
   - 폴더를 이동하는 명령어
   - cd Documents → Documents 폴더로 이동
   - cd .. → 상위 폴더로 이동
   - cd ~ → 홈 폴더로 이동

2. ls (List)
   - 현재 폴더의 파일/폴더 목록을 보여줌
   - ls -la → 숨김 파일까지 상세하게 보기

3. pwd (Print Working Directory)
   - 현재 내가 어느 폴더에 있는지 경로를 보여줌
   - /Users/이름/Documents 같은 형태로 출력

4. cat (Concatenate)
   - 파일 내용을 터미널에 출력
   - cat README.md → README.md 파일 내용 보기
   - Claude의 요약 없이 원문 그대로 볼 수 있음

5. open (Mac 전용)
   - 파일이나 폴더를 기본 앱으로 열기
   - open . → 현재 폴더를 Finder로 열기
   - 굉장히 유용한 명령어!

Claude Code에서의 활용:
- !pwd → 현재 위치 확인
- !ls → 파일 목록 보기
- !open . → Finder 열기
- !cat 파일명 → 파일 내용 원문 보기
```

## 7. 핵심 비유 모음 (강의 전체에서 일관되게 사용)

| 개념 | 비유 | 설명 |
|------|------|------|
| CLAUDE.md | 신입사원에게 주는 업무 매뉴얼 | "우리 팀은 이렇게 일해" |
| Skill | 요리 레시피 | 재료와 순서만 따라하면 같은 결과 |
| MCP | USB-C 포트 | 외부 기기를 연결하는 통로 |
| Subagent | 팀원한테 일 맡기기 | "이거 해놓고 끝나면 알려줘" |
| Agent Teams | 프로젝트 팀 | 팀원들이 서로 소통하며 협업 |
| Hook | 자동 체크리스트 / IFTTT | "이런 일이 생기면 자동으로 이거 해줘" |
| Plugin | 종합 패키지 / 앱스토어 | 여러 기능을 묶어서 한 번에 설치 |
| CLI | 텍스트 카카오톡 | 텍스트로 컴퓨터와 대화 |
| Git/GitHub | 게임 세이브 포인트 | 언제든 되돌릴 수 있음 |
